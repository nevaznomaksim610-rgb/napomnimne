# Архитектура проекта: Бот напоминаний о мероприятиях и дедлайнах

## 1. Общая идея

Система состоит из двух больших, но связанных через одну БД, подсистем:

1. **Telegram-бот (клиентская часть)** — то, что видит пользователь.
   Меню категорий → список возможностей → карточка (краткое описание + дедлайн)
   с кнопкой «Поставить напоминание». Всё внутри бота, без внешних статей.

2. **Email-агент (бэкенд-автоматизация)** — то, что работает само.
   По расписанию пишет на почты акселераторов/компаний, ждёт ответ, парсит и
   анализирует его через DeepSeek API, обновляет «этап готовности» в БД и решает,
   что показать/написать пользователю в боте.

Обе подсистемы работают над одной базой данных. Связующее звено — таблица
`opportunities` (возможности/мероприятия) и её «этап готовности» (`stage`).

```
┌──────────────────────┐         ┌──────────────────────┐
│   Telegram-бот        │         │     Email-агент       │
│  (aiogram, async)     │         │  (SMTP/IMAP + DeepSeek)│
│                       │         │                       │
│  /start → меню        │         │  1. отправка письма   │
│  категории/опции      │         │  2. ожидание ответа   │
│  карточка возможности │         │  3. парсинг IMAP      │
│  кнопка «напомнить»   │         │  4. анализ DeepSeek   │
└──────────┬───────────┘         │  5. смена stage       │
           │                     │  6. уведомление юзеру  │
           │                     └──────────┬───────────┘
           │                                │
           └───────────────┬────────────────┘
                           ▼
                  ┌──────────────────┐
                  │   База данных     │
                  │  (SQLite/Postgres)│
                  └──────────────────┘
                           ▲
                           │
                  ┌──────────────────┐
                  │   Планировщик     │
                  │  (APScheduler)    │
                  │  • рассылки       │
                  │  • проверка почты │
                  │  • напоминания    │
                  └──────────────────┘
```

## 2. Технологический стек

| Назначение            | Технология                          | Почему                                   |
|-----------------------|-------------------------------------|------------------------------------------|
| Язык                  | Python 3.13                         | Уже установлен, богатая экосистема       |
| Telegram-бот          | **aiogram 3.x** (async)             | Современный, асинхронный, удобные роутеры|

| Драйвер SQLite        | aiosqlite                           | Async доступ                             |
| Планировщик           | **APScheduler**                     | Cron-задачи: рассылки, опрос почты       |
| Отправка почты        | **aiosmtplib**                      | Async SMTP                               |
| Чтение почты          | **aioimaplib** / imaplib            | Получение ответов по IMAP                |
| Анализ ответов        | **DeepSeek API** (OpenAI-совместим) | Парсинг и классификация писем            |
| HTTP-клиент           | httpx                               | Запросы к DeepSeek                        |
| Конфиг                | pydantic-settings + .env            | Безопасное хранение токенов              |

## 4. Модель данных (таблицы)

### `opportunities` — возможности/мероприятия (ядро)
| Поле              | Тип       | Описание                                        |
|-------------------|-----------|-------------------------------------------------|
| id                | int PK    |                                                 |
| category          | enum      | accelerator / grant / ambassador                |
| title             | str       | Название (видит пользователь)                   |
| short_description | str       | Краткое описание в карточке (пара предложений)  |
| application_url   | str?      | Ссылка на форму подачи                          |
| deadline          | datetime? | Дедлайн подачи                                  |
| contact_email     | str?      | Почта для аутрича (агент пишет сюда)            |
| contact_name      | str?      | Имя/название компании                           |
| stage             | enum      | Этап готовности (см. OutreachStage)             |
| is_published      | bool      | Показывать ли пользователям в боте              |
| created_at        | datetime  |                                                 |
| updated_at        | datetime  |                                                 |

### `users` — пользователи Telegram
| id (=telegram_id) | username | first_name | created_at |

### `reminders` — подписки на напоминания
| id | user_id FK | opportunity_id FK | remind_at | offset_days | is_sent |

### `email_threads` — переписка по конкретной возможности
| id | opportunity_id FK | subject | last_sent_at | last_received_at | next_action_at | followups_sent | status |

### `email_messages` — отдельные письма (вход/выход)
| id | thread_id FK | direction (in/out) | subject | body | raw | created_at |
| analysis_json | str? | ← результат разбора DeepSeek для входящих |

## 5. «Этапы готовности» (OutreachStage) — конечный автомат

```
NEW ──► CONTACTED ──► FOLLOW_UP_1 ──► FOLLOW_UP_2 ──► (нет ответа) ──► STALLED
                 │                                        │
                 └──────────── пришёл ответ ──────────────┘
                                   │
                                   ▼
                                REPLIED
                                   │
                  ┌────────────────┼────────────────┐
                  ▼                ▼                ▼
             NEEDS_INFO       CONFIRMED          REJECTED
            (агент дописал)  (есть дедлайн/      (отказ —
                  │           условия — публ.)   закрыть)
                  ▼
             (повтор цикла, пока не CONFIRMED/REJECTED)
```

Решение о переходе принимает `analyzer.py` (DeepSeek), а сам переход выполняет
`pipeline.py`. Когда возможность переходит в `CONFIRMED`, она автоматически
публикуется (`is_published = true`) и появляется в каталоге бота.

## 6. Цикл работы Email-агента

1. **Рассылка** (job по расписанию): берём `opportunities` со `stage = NEW`
   и `contact_email != null`, отправляем первое письмо по шаблону → `CONTACTED`,
   создаём `email_thread`, ставим `next_action_at = now + N дней`.
2. **Follow-up** (job): треды, где `next_action_at` прошёл и ответа нет →
   шлём напоминание → `FOLLOW_UP_1/2`. После лимита follow-up → `STALLED`.
3. **Опрос входящих** (job): читаем IMAP, матчим письма к тредам (по адресу/теме/
   In-Reply-To), сохраняем как `email_messages(direction=in)`.
4. **Анализ** (job/триггер): для новых входящих зовём DeepSeek →
   `{intent, deadline, requirements, sentiment, next_action, summary}`.
5. **Переход этапа**: `pipeline.apply(analysis)` меняет `stage`, при необходимости
   заполняет `deadline`/`short_description`, ставит `is_published`.
6. **Уведомление**: при значимых переходах бот пишет администратору/подписчикам.

## 7. Клиентский путь (бот)

```
/start
  └─► Меню категорий:  [🚀 Акселераторы] [💰 Гранты] [🤝 Амбассадорства]
         └─► Список опций категории (только is_published):
               • <Название>  —  ⏳ до 12.07
               • <Название>  —  ⏳ до 30.08
                 └─► Карточка возможности:
                       краткое описание (пара предложений)
                       ⏳ дедлайн
                       [📝 Подать заявку] (если есть ссылка)
                       [🔔 Поставить напоминание]
                       [⬅️ Назад]
```

Напоминания: при нажатии «Поставить напоминание» создаётся `reminder` с
`remind_at = deadline - 7 дней`. Отдельный job APScheduler раз в час проверяет
`reminders` где `remind_at <= now and not is_sent` и шлёт сообщение.

## 8. Что уже сделано в этом каркасе

- Полная структура проекта и зависимости.
- Модели БД и enum-ы со всеми связями.
- Async-движок SQLite + инициализация.
- Бот: /start, меню категорий, каталог, карточка, подписка на напоминание
  (на демо-данных работает end-to-end).
- Каркас агента: SMTP/IMAP клиент, DeepSeek-анализатор, конечный автомат этапов.
- Планировщик с задачами (рассылка / follow-up / опрос почты / напоминания).
- seed_data.py с примерами, чтобы запустить и потыкать бота сразу.
